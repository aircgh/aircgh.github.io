1) What this is

opentrash-cng is an open source Python package for analyzing residential solid waste collection operations using GPS telematics, enterprise waste-management data, and municipal parcel layers. It produces RouteView — a self-contained interactive HTML map — for any route on any day, and accumulates service patterns over time to surface operational insight.

The core idea: the same tools that enterprise fleet software sells for six figures should be available to any public agency that runs residential pickup, built on open standards and open data, runnable on a laptop.

Not here to reinvent. This project is rooted in years of operational practice. The goal is to package that practice cleanly, teach it, and let other agencies benefit from it without starting from scratch.

2) The two things it does

A

RouteView

Given one route and one day, produce a polished interactive HTML map showing the route polygon, the serving vehicle’s GPS trail, landfill tips, parcels served vs missed, and tonnage delivered. Runs in seconds. Embeddable in Streamlit, FastAPI, notebooks, or a batch job that writes hundreds of HTMLs to a folder.

B

Patterns

Given a long GPS history, detect each parcel’s weekly and biweekly service signature — typical day, hour, vehicle, regularity. Produces a portable parquet file that RouteView can optionally consume to compare today’s operation against the yearly pattern.

RouteView runs with or without patterns. Patterns runs on its own cadence. Neither requires the other. Agencies with small GPS histories use RouteView alone; those with years of data layer patterns on top.

3) Who this is for

  • Public agencies running residential waste collection — cities, counties, districts, tribes.
  • Researchers studying municipal operations, environmental services, or fleet efficiency.
  • Contributors — engineers, data scientists, GIS folks who want to build on a working foundation.
  • Students and learners working through the lesson series to learn applied geospatial engineering.

4) Before you start

If you’re joining this project — as a contributor, collaborator, or learner — these three items get you grounded. Plan roughly 75 minutes of focused time, spread across the week.

  1. 1

    Understand the domain ~15 min

    Watch the GIS Day 2024 presentation “GIS for Solid Waste Management” (linked in Resources). You’ll learn what a residential pickup operation actually looks like, the three-tier GIS data taxonomy (operational, support, strategic), and the auto vs manual vehicle distinction. Short, dense, no fluff.

  2. 2

    See the product in action ~15 min

    Watch the RouteView live demo segment (first ~17 min) of the ASPRS SDSU Chapter presentation on cloud-native geospatial integration. You’ll see the actual tool running on real data — parquet compression, DuckDB speed, MapLibre HTML output, GitHub Pages deployment. This is what you’ll be building.

  3. 3

    Set up your development environment ~45 min

    Install Python 3.11 (miniforge), VS Code with Python and Jupyter extensions, git, and UV. Sign up for a GitHub account if you don’t have one. The internal prep sheet has step-by-step commands for Mac, Windows, and Linux. If you prefer watching to reading, Dr. Wu’s Geographic Software Design lectures 2 and 3 walk through the same setup visually.

That’s the prep. Anything deeper we’ll build together. You don’t need to understand the code before we start — understanding it together is the point.

7) How we work

Be curious Ask early Ship small Write it down Help each other
  • Cadence: regular weekly meeting. Async the rest of the week.
  • Questions: bring them to the weekly, post them between meetings, or schedule time — whatever gets you unblocked fastest.
  • Preparation matters. Show up having tried the thing. It’s fine to be stuck; it’s not fine to be absent from your own learning.
  • Honor code. Credit your sources. Attribute code you adapt. Be honest about what you built and what you borrowed. This is an open source project — the ethics are the craft.
  • Expect change. This is a working product being shaped in real time. Specs will shift. That’s not chaos; that’s engineering.
